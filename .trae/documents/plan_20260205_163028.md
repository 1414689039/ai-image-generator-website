# 手机端界面优化设计方案

## 1. 顶部导航 (Header) 优化
- **问题**: 当前导航菜单在手机端 (`md:hidden`) 几乎不可见，缺乏汉堡菜单，且下拉菜单样式在手机端可能溢出。
- **方案**:
  - 增加汉堡菜单按钮 (仅在移动端显示)。
  - 点击汉堡菜单滑出侧边抽屉或全屏覆盖菜单，包含：
    - 首页、灵感社区链接
    - 积分显示与充值入口
    - 用户信息与退出登录
    - 联系客服、修改密码等功能
  - 优化 Logo 和品牌展示，确保在小屏幕上紧凑。

## 2. 首页 (Home) 布局重构
- **问题**: 当前采用 Flex 布局，手机端 `GeneratePanel` 高度固定为 55%，`PreviewArea` 为 45%。这种分割可能导致输入区域太小无法滚动，或者预览区域太小看不清图。且历史记录在手机端处理较为粗糙 (`md:hidden` 或重叠)。
- **方案**:
  - **采用 Tab 切换模式 (仅移动端)**:
    - 底部增加 Tab 栏：[创作] [预览] [历史]
    - **创作 Tab**: 全屏显示 `GeneratePanel`，提供舒适的表单操作体验。
    - **预览 Tab**: 全屏显示 `PreviewArea`，重点展示生成结果。生成成功后自动跳转到此 Tab。
    - **历史 Tab**: 全屏显示 `HistoryPanel`，方便回溯。
  - **优化生成体验**:
    - `GeneratePanel` 中的 "生成" 按钮固定在底部，确保随时可点。
    - 表单元素（如宽高比选择）改为横向滑动容器，节省垂直空间。

## 3. 灵感社区 (Gallery) 体验升级
- **问题**: Grid 布局在手机端单列可能导致单屏信息密度低。
- **方案**:
  - **双列瀑布流**: 手机端改为 `grid-cols-2`，减小卡片间距。
  - **精简卡片信息**: 列表页仅显示图片、点赞数和是否免费/价格，点击进入详情后再展示 Prompt 等详细信息。
  - **详情页优化**: 详情弹窗在手机端改为 **从底部弹出的 Sheet** (类似 iOS 模态)，支持下滑关闭，图片在上，信息在下可滚动。

## 4. 实施步骤
1.  **修改 `Header.tsx`**: 实现移动端汉堡菜单及抽屉逻辑。
2.  **修改 `Home.tsx`**: 引入 Tab 状态 (`activeTab`)，根据状态渲染不同组件，替代原本的 Flex 分割布局。
3.  **修改 `Gallery.tsx`**: 调整 Grid 响应式断点，优化移动端卡片样式和详情弹窗样式。

此方案旨在通过 Tab 分离关注点，解决手机屏幕空间不足的问题，同时保留桌面端的高效分栏布局。
